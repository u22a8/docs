U+22A8 scores content against **models** you define from labelled examples. A model encodes one or more **traits** — axes of judgment such as *clarity* or *intent* — and scoring returns a number per trait, a composite, and the context to interpret it. Scoring is deterministic: the same text and the same model produce the same score every time, and no language model runs at score time.

## How it works

<Steps>
  <Step title="Define a standard">
    A model is a set of traits, each taught by labelled example **samples** — not rules, prompts, or fine-tuned weights.
  </Step>
  <Step title="Train">
    Training fits each trait's geometry in embedding space and calibrates the thresholds that separate score tiers.
  </Step>
  <Step title="Score">
    Send content to the model. It returns a score per trait with confidence and headroom — deterministically, without invoking a language model.
  </Step>
</Steps>

The public catalog under `u22a8.*` ships ready-to-use models. You can also author your own through the API.
